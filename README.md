| Заголовок                 | Хто надсилає | Призначення                                    |

## Завдання 3: WebSocket фрейми — структура повідомлень

### Структура WebSocket фрейму (спрощено):
┌─────────────────────────────────────────────────────┐
│ FIN │ RSV1│ RSV2│ RSV3│       Opcode (4 біти)       │
│  1  │  0  │  0  │  0  │  1=Text, 2=Binary, 8=Close  │
├─────────────────────────────────────────────────────┤
│ MASK│              Payload Length                   │
│  1  │  7 бітів (або 16/64 біти для великих даних)   │
├─────────────────────────────────────────────────────┤
│   Masking Key (4 байти, тільки клієнт → сервер)     │
├─────────────────────────────────────────────────────┤
│                    Payload Data                     │
└─────────────────────────────────────────────────────┘

### таблиця Opcode
| Opcode | Назва        | Призначення                           |
| ------ | ------------ | ------------------------------------- |
| 0x1    | Text Frame   | Текстові дані (UTF-8)                 |
| 0x2    | Binary Frame | Бінарні дані (файли, зображення тощо) |
| 0x8    | Close Frame  | ...                                   |
| 0x9    | Ping Frame   | ...                                   |
| 0xA    | Pong Frame   | ...                                   |
